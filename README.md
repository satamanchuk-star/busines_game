# FMCG-цепочка — деловая игра (game2)

Офлайн бизнес-игра-симулятор цепочки поставок FMCG для воркшопов. До **9 команд / 36 игроков**: 4 магазина (ритейлеры), 4 производителя (поставщики), 1 перевозчик (дистрибьютор). За 4 тура команды заказывают, производят и возят товар, балансируя личную прибыль и здоровье общей цепочки.

Два формата:
- **Сетевой режим** — Node.js + WebSocket: ведущий на ноутбуке, проектор на экран, участники на своих устройствах по WiFi.
- **Демо/лаборатория** — автономные страницы в браузере без сервера (`/demo.html`, `/sim.html`).

> Правила для игроков → [`../ПРАВИЛА_ИГРЫ.md`](../ПРАВИЛА_ИГРЫ.md) · сценарий ведущего → [`../ШПАРГАЛКА_ВЕДУЩЕГО.md`](../ШПАРГАЛКА_ВЕДУЩЕГО.md) · контекст для разработки → [`../CLAUDE.md`](../CLAUDE.md)

## Быстрый старт (< 2 минут)

```bash
npm install
npm start                 # node server.js, порт 3000, пароль ведущего fmcg2024
# открыть http://localhost:3000
```

Свой порт/пароль:
```bash
PORT=3001 ADMIN_PASS=secret npm start
# или: ./start.sh 3001 secret
```

Для локальной разработки удобно выключить авторизацию:
```bash
AUTH_ENABLED=0 npm start   # без гейта — сразу любые страницы
```

## Страницы

| URL | Кто | Назначение |
|-----|-----|-----------|
| `/` (`index.html`) | все | Лендинг: вход в игру, ссылки на демо/лабораторию |
| `/login.html` | все | Гейт-ворота: заявка на доступ / код воркшопа |
| `/play.html` | ведущий + команды | Игровой интерфейс (вход: пароль = ведущий, код команды = R1…D) |
| `/live.html` | проектор | Общий экран: фазы, итоги, награды |
| `/demo.html` | — | Гид «как проходит игра» (28 сцен, автоплей) |
| `/sim.html` | — | Лаборатория: прогон сценариев, A/B, экспорт CSV, «что если» |
| `/print.html` | — | Карточки ролей и характеров для печати |

Роли при входе на `/play.html`: **Ведущий** (пароль), **Команда** (код R1–R4 / S1–S4 / D, можно вводить кириллицей Р1/П1/Д), **Проектор**.

## Конфигурация (`.env`)

Скопировать `.env.example` → `.env` (на сервере; не коммитить). Ключевые переменные:

| Переменная | Назначение |
|-----------|-----------|
| `AUTH_ENABLED` | `1` — гейт включён (по умолч.), `0` — выключить для отладки |
| `WORKSHOP_CODE` | общий код входа на воркшоп → вкладка «Код игры» на login. Снимает затор персональных заявок. После игры очистить |
| `RESEND_API_KEY` | отправка писем-заявок владельцу (Resend HTTP API); пусто → ссылка в лог |
| `OWNER_EMAIL` | куда приходят заявки на доступ |
| `BASE_URL` | для ссылок в письме (`https://ibsgame.ru`) |
| `ADMIN_PASS` | пароль ведущего (по умолч. `fmcg2024`) |
| `COOKIE_SECURE` | `1` для HTTPS-прод |

## Тесты

```bash
npm test          # smoke + engine (быстро)
npm run test:all  # + protocol + auth (полный)
```
Покрытие: `smoke` (консистентность конфига/HTML), `engine` (движок `calcRound`, здоровье, счёт), `protocol` (WebSocket), `auth` (гейт, заявки, воркшоп).

> ⚠️ `auth.test.js` поднимает сервер на :3002. Если в `test:all` упал из-за гонки порта при других локальных серверах — прогнать отдельно: `npm run test:auth`.

## Архитектура (кратко)

- **`public/gameconfig.js`** — единый источник правды (спрос, экономика, команды). Подключается и в браузере, и в Node.
- **`server.js`** — WebSocket-сервер + авторитетный движок `calcRound(round, decisions)`. Копия движка есть в `sim.html` для автономной лаборатории (при изменении логики — синхронизировать обе).
- **`auth.js`** — гейт перед статикой: заявка+одобрение (Resend), режим воркшопа, `/fonts/` открыт без авторизации.
- **`public/`** — все экраны (см. таблицу) + self-hosted шрифт Onest в `public/fonts/`.

Подробно (формулы счёта, здоровье цепочки, фазы, ловушки) → [`../CLAUDE.md`](../CLAUDE.md).

## Деплой (PM2 на VPS)

Статика отдаётся с `Cache-Control: no-cache` → после деплоя свежая версия без Ctrl+Shift+R.

```bash
# фронтенд и бэкенд — РАЗНЫМИ командами (см. предупреждение ниже)
rsync -az -e "ssh -i ~/.ssh/КЛЮЧ" public/ ПОЛЬЗОВАТЕЛЬ@ХОСТ:/var/www/fmcg-game/public/
rsync -az -e "ssh -i ~/.ssh/КЛЮЧ" server.js auth.js ПОЛЬЗОВАТЕЛЬ@ХОСТ:/var/www/fmcg-game/
ssh -i ~/.ssh/КЛЮЧ ПОЛЬЗОВАТЕЛЬ@ХОСТ 'pm2 restart fmcg-game'
```

> ⚠️ **НЕ** объединять `server.js public/` в одну rsync-команду с целью-корнем: `public/` с trailing-slash вывалит все html в корень. Бэкенд и `public/` — раздельно.
> Точные хост/путь/ключ — в [`../CLAUDE.md`](../CLAUDE.md) (раздел «Деплой»). Состояние игры (`game-state.json`) и доступы (`auth-store.json`) переживают рестарт.
